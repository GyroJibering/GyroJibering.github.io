现代爬虫对抗，除了传统的那些 ip 代理池、账号代理池之外，还有一些高级的方法：比如浏览器指纹，用于检测某些无头爬虫，利用 WebGL Fingerprint 之类的方式，可以检测出非常多的爬虫。

<!--more-->

常见的信号大概包括这些：

- navigator
- canvas
- webgl
- audio
- fonts
- plugins
- screen
- timezone
- mouse movement
- keyboard typing
- battery API
- media devices

什么是 navigator，你的意思是我可以通过 WebGL Fingerprint 这种方式来检测爬虫的伪装对吗？

你的理解已经很接近真实的反爬体系了，我把两个问题分别讲清楚。

## 什么是 navigator

navigator 是浏览器提供给 JavaScript 的一个全局对象。作用是向网页暴露浏览器和设备的一些信息。

在浏览器控制台输入：

```javascript
navigator
```

你会看到类似：

```javascript
Navigator {
  userAgent: "Mozilla/5.0 ..."
  language: "zh-CN"
  platform: "MacIntel"
  hardwareConcurrency: 8
  webdriver: false
}
```

它本质上属于 BOM（Browser Object Model）。

浏览器对象模型大致结构：

```text
window
 ├─ document
 ├─ navigator
 ├─ location
 ├─ history
 ├─ screen
```

常见 navigator 属性。

User Agent：

```javascript
navigator.userAgent
```

语言：

```javascript
navigator.language
navigator.languages
```

CPU 核心数：

```javascript
navigator.hardwareConcurrency
```

内存：

```javascript
navigator.deviceMemory
```

插件：

```javascript
navigator.plugins
```

自动化检测里最关键的一个：

```javascript
navigator.webdriver
```

如果返回 true，说明浏览器是 Selenium / Puppeteer / Playwright 自动化控制。

## WebGL Fingerprint 能检测爬虫伪装吗？

答案是：可以检测，但不是直接检测。它的作用是检测浏览器环境是否真实。

真实浏览器运行 Canvas / WebGL / Audio，会产生稳定但设备相关的结果。例如 WebGL 查询：

```javascript
gl.getParameter(gl.RENDERER)
```

可能得到：

```text
ANGLE (Apple, Apple M1, Metal)
```

或者：

```text
ANGLE (NVIDIA RTX 3080 Direct3D11)
```

不同设备渲染结果不同。

很多爬虫运行在无头 Chrome、Docker、虚拟机、服务器。问题是服务器通常没有 GPU，WebGL 会返回 SwiftShader，例如：

```text
ANGLE (Google SwiftShader)
```

这非常典型：软件渲染 / 虚拟 GPU。所以网站会判断：如果 renderer == SwiftShader → 很可能是爬虫。

Canvas 指纹也能检测。网站会画一张图：

```javascript
ctx.fillText("hello", 10, 10);
```

然后：

```javascript
canvas.toDataURL()
```

得到图片 hash。如果大量用户都出现同一个 canvas hash，说明来自同一个虚拟环境，通常是 headless chrome。

真正的反爬逻辑不会只用一种检测，通常会组合 navigator 指纹 + canvas + webgl + audio + 字体 + 鼠标轨迹 + TLS fingerprint。

例如：

```text
score = 0
if webdriver → +40
if swiftshader → +20
if plugins=0 → +10
if no mouse move → +30
score > 50 → bot
```

为什么爬虫要伪造 navigator？因为很多网站会检查 `navigator.webdriver`，所以 stealth 插件会改写：

```javascript
Object.defineProperty(navigator, "webdriver", {
  get: () => false
});
```

但问题来了：伪装 navigator 很容易；伪装 Canvas / WebGL / Audio / GPU / 字体就非常困难，因为这些涉及图形驱动、系统字体、GPU 渲染，这也是浏览器指纹难以伪造的原因。

真实反爬的一个经典检测，很多网站会这样检测：

```javascript
if (navigator.webdriver) bot = true;
if (gl.getParameter(gl.RENDERER).includes("SwiftShader")) bot = true;
if (navigator.plugins.length === 0) bot = true;
```

三条同时满足，99% 是自动化。

一句话总结：navigator 是浏览器提供给 JS 的浏览器信息接口。WebGL Fingerprint 的作用不是直接检测爬虫，而是通过 GPU 渲染特征判断浏览器环境是否真实，从而识别自动化环境。

如果你继续深入前端安全，我强烈建议你了解一个非常关键的技术：“Browser Environment Consistency Check（浏览器环境一致性检测）”。这是目前 Cloudflare / Akamai 检测爬虫最核心的技术，很多高级爬虫都会死在这里。

那 WebGL Fingerprint 这种方式岂不是可以解决掉 99.99% 的爬虫了？

你的这个直觉其实是很多刚接触反爬的人都会有的想法，但现实情况是：WebGL Fingerprint 很强，但绝对不可能解决 99.99% 的爬虫。原因在于：WebGL 指纹只是在客户端环境检测体系中的一环，而不是决定性手段。

WebGL 指纹利用的是 GPU 渲染差异，典型代码：

```javascript
var canvas = document.createElement("canvas");
var gl = canvas.getContext("webgl");

var vendor = gl.getParameter(gl.VENDOR);
var renderer = gl.getParameter(gl.RENDERER);
```

这确实非常难伪造。但高级爬虫不会直接用服务器环境，他们会用真实浏览器环境，例如云手机 / 云真机（GPU 是真的），或者 Windows VPS + GPU 运行 Chrome GUI（WebGL 完全真实），甚至可以用 `--use-gl=desktop` 调用 GPU，还可以 hook `getParameter` 做 WebGL spoof。

真正的问题仍然是一致性检测。反爬系统不是看单个特征，而是看特征之间是否一致，例如：

```text
GPU: RTX 4090
CPU cores: 1
Memory: 1GB
screen: 800x600
plugins: 0
```

明显不合理，这叫 Browser Environment Consistency Check。

真实反爬系统会检测很多维度，例如 navigator / canvas / webgl / audio / fonts / plugins / screen / timezone / mouse movement / keyboard typing / battery API / media devices，然后做一个 risk score，例如 score > 70 → bot。

更强的一层是行为分析，例如鼠标轨迹、滚动速度、点击延迟、页面停留时间。还有网络层指纹：TLS handshake、TCP fingerprint、HTTP2 settings，例如 JA3 fingerprint。不同客户端（Chrome / curl / python requests / nodejs）TLS 参数完全不同。

真实世界反爬是多层体系：JS 指纹 + 行为分析 + 网络指纹 + IP 信誉 + 账号行为共同判断，WebGL 只是其中一个信号。

为什么很多人高估 WebGL？因为在低级爬虫场景（python requests / selenium / 简单 puppeteer）WebGL 一下就能抓出来，但面对专业爬虫公司，WebGL 几乎不起作用。

一句话总结：WebGL Fingerprint 的作用是检测浏览器 GPU 渲染特征，用于识别虚拟环境，但它只能识别低级自动化环境。真正的反爬体系依赖浏览器指纹 + 行为分析 + 网络指纹 + 风控模型。
